Subsystem 01
CustomNet121 (Chest X-Ray)
  • Architecture: A heavily fine-tuned, custom implementation of DenseNet121.
  • Dataset Pipeline: Formally trained on the massive NIH ChestX-ray8 Dataset encompassing 112,120 labeled patient radiograph images.
  • Code Structure: Engineered in finetune_chest_model.py mapping 14 pathologies via ImageDataGenerator with manual Cosine Decay learning schedules.
  • Verification: Backed by raw logs inside chest_finetune_history.csv proving local programmatic execution lowering loss to 0.153 natively over 10 epochs.
Epoch 10 Validation Accuracy 94.4%

Subsystem 02
InceptionV3 (Brain Tumor MRI)
  • Model Architecture: Base InceptionV3 architecture structured with custom fully-connected dense classification heads.
  • Code Implementation: The entire data ingestion, image padding, and mapping sequence is demonstrated directly in braintTumor.ipynb.
  • Training Origin: Ingests the Brain Tumor Classification MRI Dataset consisting of 7,023 distinct slices.
  • Classification Vectors: Accurately categorizes into strict classes: Glioma, Meningioma, Pituitary tumor, or Healthy no tumor.

Subsystems 03 & 04
Zero-Shot Transformers & Validation

🦴 MedSigLIP (Orthopedics)

Architecture: Advanced HuggingFace Vision-Language Transformer model.

The Approach: Employs Zero-Shot Classification. Instead of training fixed image classifiers for every bone, it dynamically pairs visual tensors with 20+ text pathology labels at runtime to detect fractures instantly.

📋 CheXpert Parallel Layer

Dataset: Originated from Stanford's CheXpert array (224,316 images).

The Approach: Acts as an integrated parallel validation mechanism alongside CustomNet121. It strictly calculates logits across 14 localized chest pathologies to output the precise visual bar probabilities on the frontend UI.

08 / 13
Subsystem 05
Groq NLP & Dynamic PDF Engineering
  • The Accelerator: Bypassed standard, slow OpenAI APIs to leverage the revolutionary **Groq LPU (Language Processing Unit)** running LLaMA-3.3-70b.
  • Speed Implementation: It ingests PyTorch array confidences and constructs highly empathetic, structurally accurate 1,500-token medical reports in less than two seconds.
  • Custom Edge Fine-Tuning: Built a native localized pipeline (fine_tune_llm.py). We demonstrated the ability to wrap HuggingFace AutoModelForCausalLM directly with PyTorch Data Collators to fine-tune `google/medgemma-27b-it` over structured JSON datasets.
Proves that NeuroVision's architecture isn't permanently tied to cloud APIs. It is engineered to host fully local, edge-enabled natural language generative inferences if internet connectivity drops in remote clinics.
09 / 13
Scale & Reach (1/2)
The Comprehensive Product Ecosystem

1. NeuroVision Core Diagnostics

The nexus of the application. Handles drag-and-drop file ingestion, triggers pipeline routing, visualizes exact inference probability bars, and compiles the downloadable ReportLab PDF.

2. Embedded MedBot Context Engine

A 24/7 autonomous symptom checker natively floating on the patient dashboard, designed as a highly empathetic first-layer triage protocol before formal diagnosis.

3. Aria (Vocal Therapist Interface)

Taps the native browser Web Speech API and Groq audio translation to provide interactive spoken-word psychological grounding sessions without keyboard barriers.

4. Clinical Mental Health Planner

Digitizes globally recognized clinical scales: PHQ-9 (depression screening) and GAD-7 (anxiety indexing), producing actionable routine timelines based on score thresholds.

5. Neurological Baseline Game Tracking
